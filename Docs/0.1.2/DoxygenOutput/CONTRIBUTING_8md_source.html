2 **Welcome! The purpose of this document is to make the contributing process as

6 ## General Behavior
7 * We do not tolerate racist language.
8 * Refrain from arguing about religion.
9 * Do not attempt to force your opinions on others (please note that this
10 specifies *opinions* not facts. This does not apply to correcting factual
11 inaccuracies)

19 ## Steps to Contribute
20 
21 
22 ### First Submit a Proposal
23 Before starting work on a new feature or major bugfix make sure to submit a
24 proposal in the form of a Github bug report. Explain the feature or bugfix you
25 wish to add or fix and give any information on how you plan on doing so. This
26 is very important as it makes sure that the scope of work and the implementation
27 are good before work is started. Otherwise the merge might have to be rejected
28 which is rather disheartening for all parties involved.
29 
30 
31 ### Then Start Work
32 Once the proposal has been approved and the approach deemed sound work can
33 begin. While working make sure to keep a tidy commit history with well sized
34 commits which have short descriptive names. Of course it goes without saying
35 that one should work to produce high quality code. If you have any further
36 questions while working, the proposal bug and the SkyofSteel Discord are both
37 good placed to ask, depending on the importance and time sensitivity of said
38 questions.
39 
40 
41 ### Merge Time
42 When the implementation is finished it is time to submit a PR. Before doing this
43 make sure that the git log is clean-n-tidy and that only required changes have
44 been made. In the PR message make sure to link to the original proposal.
